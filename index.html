About the Events

Code Battle - A Mini Hackathon

The Mini Hackathon ‚Äì Code Battle is a competitive programming event divided into three progressive levels designed to test participants‚Äô problem-solving skills, coding agility, and ability to create real-world impact simulations. Participants will be evaluated based on accuracy, speed, creativity, and relevance. The entire event is scored out of 100 marks, distributed across the three levels.

Code Arena - A Python 2D Game Development Workshop

Code Arena is a hands-on workshop on 2D Game Development using Python. This activity is designed to introduce students to the fundamentals of game design and programming through the popular Python library Pygame. Participants will learn how to create interactive 2D games by coding player movement, handling collisions, designing levels, and adding sound/graphics. By the end of the workshop, participants will have a working mini-game and a strong foundation to build more advanced projects in the future.

  • Introduce students to Python-based 2D game development.
  • Provide practical experience with Pygame library and game loops.
  • Teach concepts like player movement, events, collision detection, and scoring.
  • Encourage creativity, problem-solving, and design thinking through game projects.
  • Build confidence to design and deploy small games independently.

Rules & Guidelines

Code Battle Rules

General Rules

  • Eligibility: Open to all students, developers, and tech enthusiasts with basic coding knowledge.
  • Team Composition: Participants may compete individually or in teams of up to 4 members.
  • Registration: All participants must register before the event deadline.
  • Laptop Requirement: Participants must bring their own laptops with necessary software installed and Bring charger or fully¬†charged Laptop
  • Code of Conduct: All participants are expected to maintain professionalism and refrain from plagiarism, cheating, or disruptive behavior.
  • Tools and Languages: C & Python is allowed unless specified otherwise. Internet access may be restricted during certain rounds.
  • Disqualification: Participants may be disqualified for unethical practices or not adhering to guidelines.

General Guidelines

  • All submissions must be original.
  • Code must be properly commented where applicable.
  • Use of public resources (like AI tools) is allowed in Level 2 & 3.
  • Participants must submit their presentation before the deadline.
  • Final decisions by the judges are binding.

Code Arena Rules

Participation & Technology

Project Rules

Developed only during the workshop. Students must submit or showcase their mini-game at the end.

Code Battle Scoring Breakdown

Level 1: Code Quest

Algorithmic problem-solving round.

30 Marks

  • Objective: Solve algorithmic problems within 90 minutes.
  • Scoring: Full, partial, or no points based on solution correctness.

Level 2: Livewire Challenge

Dynamic problem-solving under time constraints.

30 Marks

  • Objective: Solve dynamically assigned problems in 60 minutes.
  • Scoring: Full points for correct solution, with bonus points for fastest submissions.

Level 3: Impact Forge

Community impact simulation prototype.

40 Marks

  • Objective: Design and build a simulation for a real-world problem.
  • Scoring: 10 marks each for Innovation, Practicality, Code Quality, and Presentation.
